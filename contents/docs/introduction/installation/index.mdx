## Requisitos previos

<Step>
<StepItem title="Requisitos del sistema">

- **Node.js:** Asegúrate de tener instalado Node.js. El proyecto utiliza `npm`, que viene con Node.js. Instala Node.js desde aquí.
- **NPM or Yarn:** Dado que el proyecto utiliza comandos `npm`, asegúrese de que npm esté instalado con Node.js. También puede utilizar Yarn si lo prefiere.

Para verificar la instalación, ejecute:

```bash
node -v
npm -v
```

</StepItem>
<StepItem title="Git">
Necesitas tener Git instalado para clonar el repositorio. Descarga Git desde [aquí](https://git-scm.com/).

```bash
git --version
```

</StepItem>
<StepItem title="Editor de código">
Necesitará un editor de código para trabajar en el proyecto. Las opciones más comunes incluyen:

- [Visual Studio Code](https://code.visualstudio.com/)
- [WebStorm](https://www.jetbrains.com/webstorm/)
- Cualquier otro editor de texto que prefieras.

</StepItem>
<StepItem title="Conocimiento requerido">
- **Next.js:** Dado que este es un proyecto de Next.js, debes estar familiarizado con los conceptos básicos de Next.js.
- **React:** El proyecto utiliza React, por lo que comprender la estructura de componentes y los hooks de React será útil.
- **TypeScript:** El proyecto está escrito en TypeScript, por lo que debes tener un conocimiento básico del mismo.
- **Tailwind:** El proyecto utiliza Tailwind CSS para el estilo.
- **MDX:** Se utiliza Markdown + JSX (MDX) para escribir documentación.

</StepItem>
</Step>

## Instalación

<Step>
<StepItem title="Clonar el repositorio">
Primero, clona el repositorio en tu máquina local usando Git:

```bash
git clone https://github.com/cjhirashi/documents.git
```

Cambie el directorio a la carpeta del proyecto:

```bash
cd rubix-documents
```

</StepItem>
<StepItem title="Install Dependencies">
Use npm to install all the required dependencies:

```bash
npm install
```

This will install all the dependencies listed in the package.json file, including:

- Next.js
- React
- TypeScript
- Tailwind CSS
- MDX components
- Other dependencies like ESLint, PostCSS, Husky, etc.

</StepItem>

<StepItem title="Running the Development Server">
Start the development server by running:

```bash
npm run dev
```

This will start the local server, and you can open your browser at:

```arduino
http://localhost:3000
```

</StepItem>

<StepItem title="Editing the Project">
  - **MDX Files:** Documentation content is stored in `contents/docs`.
  - **Components:** You can edit React components in the `components/` directory.
  - **Tailwind:** Tailwind styling are managed in `tailwind.config.ts` and `app/globals.css`.

To make any changes to the project, open the files in your editor and modify them as needed.

<Note title="Required" type="danger">
  The project's search functionality relies on the Husky's automation to build
  `search-data/documents.json` ensure git commit is performed to generate this
  file.
</Note>

</StepItem>
<StepItem title="Building for Production">
Once you’ve made your changes and want to build the project for production:

```bash
npm run build
```

This will generate a production-ready build of the app.

</StepItem>
<StepItem title="Running Production Server">
To run the production server:

```bash
npm run start
```

The production app will be available at:

```arduino
http://localhost:3000
```

</StepItem>
</Step>
