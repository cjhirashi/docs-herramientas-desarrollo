## Requisitos previos

<Step>
<StepItem title="Requisitos del sistema">

- **Node.js:** Asegúrate de tener instalado Node.js. El proyecto utiliza `npm`, que viene con Node.js. Instala Node.js desde aquí.
- **NPM or Yarn:** Dado que el proyecto utiliza comandos `npm`, asegúrese de que npm esté instalado con Node.js. También puede utilizar Yarn si lo prefiere.

Para verificar la instalación, ejecute:

```bash
node -v
npm -v
```

</StepItem>
<StepItem title="Git">
Necesitas tener Git instalado para clonar el repositorio. Descarga Git desde [aquí](https://git-scm.com/).

```bash
git --version
```

</StepItem>
<StepItem title="Editor de código">
Necesitará un editor de código para trabajar en el proyecto. Las opciones más comunes incluyen:

- [Visual Studio Code](https://code.visualstudio.com/)
- [WebStorm](https://www.jetbrains.com/webstorm/)
- Cualquier otro editor de texto que prefieras.

</StepItem>
<StepItem title="Conocimiento requerido">
- **Next.js:** Dado que este es un proyecto de Next.js, debes estar familiarizado con los conceptos básicos de Next.js.
- **React:** El proyecto utiliza React, por lo que comprender la estructura de componentes y los hooks de React será útil.
- **TypeScript:** El proyecto está escrito en TypeScript, por lo que debes tener un conocimiento básico del mismo.
- **Tailwind:** El proyecto utiliza Tailwind CSS para el estilo.
- **MDX:** Se utiliza Markdown + JSX (MDX) para escribir documentación.

</StepItem>
</Step>

## Instalación

<Step>
<StepItem title="Clonar el repositorio">
Primero, clona el repositorio en tu máquina local usando Git:

```bash
git clone https://github.com/cjhirashi/documents.git
```

Cambie el directorio a la carpeta del proyecto:

```bash
cd documents
```

</StepItem>
<StepItem title="Instalar dependencias">
Utilice npm para instalar todas las dependencias necesarias:

```bash
npm install
```

Esto instalará todas las dependencias enumeradas en el archivo package.json, incluidas:

- Next.js
- React
- TypeScript
- Tailwind CSS
- MDX components
- Otras dependencias como ESLint, PostCSS, Husky, etc.

</StepItem>

<StepItem title="Ejecutando el servidor de desarrollo">
Inicie el servidor de desarrollo ejecutando:

```bash
npm run dev
```

Esto iniciará el servidor local y podrás abrir tu navegador en:

```arduino
http://localhost:3000
```

</StepItem>

<StepItem title="Editando el proyecto">
  - **Archivos MDX:** El contenido de la documentación se almacena en `contents/docs`.
  - **Components:** Puedes editar componentes de React en el directorio `components/`.
  - **Tailwind:** El estilo de Tailwind se gestiona en `tailwind.config.ts` y `app/globals.css`.

Para realizar cualquier cambio en el proyecto, abra los archivos en su editor y modifíquelos según sea necesario.

<Note title="Requerido" type="danger">
  La funcionalidad de búsqueda del proyecto se basa en la automatización de Husky para crear
`search-data/documents.json` y garantizar que se realice la confirmación de Git para generar este archivo.
</Note>

</StepItem>
<StepItem title="Contrucción en producción">
Una vez que haya realizado los cambios y desee crear el proyecto para producción:

```bash
npm run build
```

Esto generará una compilación de la aplicación lista para producción.

</StepItem>
<StepItem title="Ejecución del servidor de producción">
Para ejecutar el servidor de producción:

```bash
npm run start
```

La aplicación de producción estará disponible en:

```arduino
http://localhost:3000
```

</StepItem>
</Step>

## Opcional

<Step>
<StepItem title="Implementación en Vercel">
Este proyecto está optimizado para ser implementado en Vercel. Para implementarlo en Vercel:

- Ir a [vercel.com](https://vercel.com/) y regístrate.
- Vincula tu repositorio de GitHub e implementa el proyecto.
- Vercel se encargará del proceso de construcción de forma automática.
</StepItem>
<StepItem title="Hooks de Pre-commit y Linting">
El proyecto utiliza ESLint para el análisis de errores y Husky para los hooks de Git. Estas herramientas se configuran automáticamente, pero puedes personalizarlas en los directorios `.eslintrc.json` y `.husky`.
</StepItem>
</Step>

## Important Information

La funcionalidad de búsqueda del proyecto se basa en la automatización de Husky para crear `search-data/documents.json` y garantizar que se realice la confirmación de Git para generar este archivo.
