Los formularios en HTML son una herramienta fundamental para la interacción entre los usuarios y los sitios web. Permiten recolectar información valiosa, como datos de contacto, comentarios, preferencias y cualquier otro tipo de entrada que el sitio necesite. Un formulario bien diseñado mejora la experiencia del usuario y facilita la gestión de la información.

## OBJETIVOS CLAVE

*   **Recolección de Datos:** Permitir a los usuarios ingresar y enviar datos al servidor.
*   **Validación de Datos:** Asegurar que los datos ingresados sean correctos y cumplan con los requisitos del sitio.
*   **Interactividad:** Ofrecer una experiencia de usuario intuitiva y fácil de usar.
*   **Accesibilidad:** Diseñar formularios que sean accesibles para todos los usuarios, incluyendo aquellos con discapacidades.

## ETIQUETA `<FORM>`

El elemento `<form>` es el contenedor principal de todos los elementos de un formulario. Define el formulario y especifica cómo se enviarán los datos al servidor.

```html
<form action="/procesar_formulario" method="post">
  <!-- Elementos del formulario aquí -->
</form>
```

*   **`action`:** Especifica la URL a la que se enviarán los datos del formulario.
*   **`method`:** Define el método HTTP utilizado para enviar los datos (generalmente `get` o `post`).

## ELEMENTOS DE ENTRADA `<INPUT>`

El elemento `<input>` es el más común en los formularios y se utiliza para crear diferentes tipos de campos de entrada.

### TIPOS DE INPUT

*   **`text`:** Para campos de texto de una sola línea.

    ```html
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">
    ```

*   **`password`:** Para campos de contraseña, donde el texto se oculta.

    ```html
    <label for="password">Contraseña:</label>
    <input type="password" id="password" name="password">
    ```

*   **`email`:** Para campos de correo electrónico, con validación básica del formato.

    ```html
    <label for="email">Correo Electrónico:</label>
    <input type="email" id="email" name="email">
    ```

*   **`number`:** Para campos numéricos, con opciones para definir valores mínimos y máximos.

    ```html
    <label for="edad">Edad:</label>
    <input type="number" id="edad" name="edad" min="18" max="99">
    ```

*   **`date`:** Para seleccionar una fecha del calendario.

    ```html
    <label for="fecha_nacimiento">Fecha de Nacimiento:</label>
    <input type="date" id="fecha_nacimiento" name="fecha_nacimiento">
    ```

*   **`radio`:** Para opciones de selección única.

    ```html
    <p>Género:</p>
    <input type="radio" id="masculino" name="genero" value="masculino">
    <label for="masculino">Masculino</label><br>
    <input type="radio" id="femenino" name="genero" value="femenino">
    <label for="femenino">Femenino</label><br>
    <input type="radio" id="otro" name="genero" value="otro">
    <label for="otro">Otro</label>
    ```

*   **`checkbox`:** Para opciones de selección múltiple.

    ```html
    <p>Intereses:</p>
    <input type="checkbox" id="musica" name="intereses" value="musica">
    <label for="musica">Música</label><br>
    <input type="checkbox" id="deporte" name="intereses" value="deporte">
    <label for="deporte">Deporte</label><br>
    <input type="checkbox" id="cine" name="intereses" value="cine">
    <label for="cine">Cine</label>
    ```

*   **`file`:** Para permitir la carga de archivos.

    ```html
    <label for="archivo">Subir Archivo:</label>
    <input type="file" id="archivo" name="archivo">
    ```

*   **`submit`:** Para crear un botón que envía el formulario.

    ```html
    <input type="submit" value="Enviar">
    ```

*   **`reset`:** Para crear un botón que resetea el formulario a sus valores predeterminados.

    ```html
    <input type="reset" value="Restablecer">
    ```

## ETIQUETA `<TEXTAREA>`

El elemento `<textarea>` se utiliza para crear campos de texto de múltiples líneas, ideales para comentarios o descripciones.

```html
<label for="comentarios">Comentarios:</label><br>
<textarea id="comentarios" name="comentarios" rows="4" cols="50"></textarea>
```

*   **`rows`:** Especifica el número de filas visibles en el área de texto.
*   **`cols`:** Especifica el número de columnas visibles en el área de texto.

## ETIQUETA `<SELECT>`

El elemento `<select>` crea una lista desplegable de opciones.

```html
<label for="pais">País:</label>
<select id="pais" name="pais">
  <option value="mexico">México</option>
  <option value="colombia">Colombia</option>
  <option value="espana">España</option>
  <option value="argentina">Argentina</option>
</select>
```

*   **`<option>`:** Define cada una de las opciones en la lista desplegable.

## ETIQUETA `<LABEL>`

El elemento `<label>` proporciona una descripción para los elementos del formulario, mejorando la accesibilidad y la usabilidad.

```html
<label for="nombre">Nombre:</label>
<input type="text" id="nombre" name="nombre">
```

Al asociar un `<label>` con un elemento de formulario mediante el atributo `for`, se mejora la accesibilidad, ya que los usuarios pueden hacer clic en la etiqueta para enfocar el elemento asociado.

## ATRIBUTOS IMPORTANTES

*   **`name`:** Especifica el nombre del elemento, que se utiliza para identificar los datos cuando se envían al servidor.
*   **`id`:** Proporciona un identificador único para el elemento, útil para CSS y JavaScript.
*   **`value`:** Define el valor inicial del elemento.
*   **`placeholder`:** Muestra un texto de ejemplo dentro del campo hasta que el usuario ingrese algo.
*   **`required`:** Indica que el campo es obligatorio y debe ser llenado antes de enviar el formulario.
*   **`disabled`:** Desactiva el campo, impidiendo que el usuario interactúe con él.
*   **`readonly`:** Permite que el usuario vea el valor del campo, pero no lo modifique.

## VALIDACION DE FORMULARIOS

La validación de formularios es crucial para asegurar que los datos ingresados por los usuarios sean correctos y cumplan con los requisitos del sitio.

### VALIDACION HTML5

HTML5 proporciona atributos de validación integrados, como `required`, `min`, `max`, `pattern` y `type`.

```html
<input type="email" id="email" name="email" required>
<input type="number" id="edad" name="edad" min="18" max="99">
<input type="text" id="codigo_postal" name="codigo_postal" pattern="[0-9]{5}">
```

### VALIDACION CON JAVASCRIPT

Para una validación más compleja, se puede utilizar JavaScript. Esto permite realizar comprobaciones personalizadas y mostrar mensajes de error específicos.

```javascript
const formulario = document.getElementById('miFormulario');

formulario.addEventListener('submit', function(event) {
  let email = document.getElementById('email').value;
  if (!email.includes('@')) {
    alert('Por favor, introduce un correo electrónico válido.');
    event.preventDefault(); // Evita que el formulario se envíe
  }
});
```

## ACCESIBILIDAD EN FORMULARIOS

Es fundamental diseñar formularios que sean accesibles para todos los usuarios, incluyendo aquellos con discapacidades.

*   **Etiquetas:** Utilizar etiquetas `<label>` para todos los elementos del formulario.
*   **Atributo `aria-label`:** Proporcionar descripciones adicionales para elementos complejos.
*   **Estructura lógica:** Organizar los elementos del formulario de manera lógica y coherente.
*   **Validación accesible:** Asegurarse de que los mensajes de error sean claros y fáciles de entender.
