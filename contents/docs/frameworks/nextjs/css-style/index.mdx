## COMO USAR CSS EN TU APLICACION 

***Next.js*** ofrece varias formas de usar ***CSS*** en su aplicación, entre ellas:

- [CSS Modules](https://nextjs.org/docs/app/getting-started/css-and-styling#css-modules)
- [Global CSS](https://nextjs.org/docs/app/getting-started/css-and-styling#global-css)
- [Tailwind CSS](https://nextjs.org/docs/app/getting-started/css-and-styling#tailwind-css)
- [Sass](https://nextjs.org/docs/app/getting-started/css-and-styling#sass)
- [CSS-in-JS](https://nextjs.org/docs/app/getting-started/css-and-styling#css-in-js)
- [External Stylesheets](https://nextjs.org/docs/app/getting-started/css-and-styling#external-stylesheets)

Esta página le guiará sobre cómo utilizar cada uno de estos enfoques.

## MODULOS CSS

Los módulos ***CSS*** controlan localmente el ***CSS*** generando nombres de clase únicos. Esto le permite usar la misma clase en distintos archivos sin preocuparse por colisiones.

Para comenzar a utilizar módulos ***CSS***, cree un nuevo archivo con la extensión `.module.css` e impórtelo en cualquier componente dentro del directorio de la aplicación:

`app/blog/styles.module.css`

```css
.blog {
    padding: 24px;
}
```

`app/blog/page.tsx`

```tsx
import styles from './styles.module.css'

export default function Page({ children }: { children: React.ReactNode }) {
    return <main className={styles.blog}>{children}</main>
}
```

## CSS global

Puede utilizar ***CSS*** global para aplicar estilos en toda su aplicación.

Para utilizar estilos globales, cree un nuevo archivo ***CSS***, por ejemplo `app/global.css`:

app/global.css

```css
body {
    padding: 20px 20px 60px;
    max-width: 680px;
    margin: 0 auto;
}
```

Importa el archivo en el diseño raíz (`app/layout.js`) para aplicar los estilos a cada ruta en tu aplicación:

`app/layout.js`

```tsx
// These styles apply to every route in the application
import './global.css'

export default function RootLayout({
    children,
}: {
    children: React.ReactNode
}) {
    return (
        <html lang="en">
            <body>{children}</body>
        </html>
    )
}
```

<aside>
ℹ️

Bueno saber:

Los estilos globales se pueden importar a cualquier ***layout***, ***pages*** o ***componente*** dentro del directorio de la aplicación. Sin embargo, ***Next.js*** utiliza el soporte integrado de ***React*** para hojas de estilo para integrarse con Suspense. Este soporte integrado actualmente no elimina las hojas de estilo mientras navega entre rutas. Por lo tanto, recomendamos usar estilos globales para ***CSS*** verdaderamente global y módulos ***CSS*** para ***CSS*** con alcance.

</aside>

## TAILWIND CSS

***Tailwind CSS*** es un marco ***CSS*** que prioriza la utilidad y que se integra perfectamente con ***Next.js***.

### INSTALACION DE TAILWIND 

Para comenzar a usar ***Tailwind***, instale los paquetes ***CSS*** de ***Tailwind*** y ejecute el comando `init` para generar los archivos `tailwind.config.js` y `postcss.config.js`:

```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

### CONFIGURACION DE TAILWIND 

Dentro del archivo de configuración de ***Tailwind***, agregue rutas a los archivos que usarán los nombres de clase de ***Tailwind***:

`tailwind.config.ts`

```tsx
import type { Config } from 'tailwindcss'

export default {
    content: [
        './app/**/*.{js,ts,jsx,tsx,mdx}',
        // Or if using `src` directory:
        './src/**/*.{js,ts,jsx,tsx,mdx}',
    ],
    theme: {
        extend: {},
    },
    plugins: [],
} satisfies Config
```

### Usando Tailwind

Agregue las directivas ***Tailwind*** a su hoja de estilo global:

`app/globals.css`

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

Luego, importe los estilos en el diseño raíz:

`app/layout.tsx`

```tsx
import type { Metadata } from 'next'

// These styles apply to every route in the application
import './globals.css'

export const metadata: Metadata = {
    title: 'Create Next App',
    description: 'Generated by create next app',
}

export default function RootLayout({
    children,
}: {
    children: React.ReactNode
}) {
    return (
        <html lang="en">
            <body>{children}</body>
        </html>
    )
}
```

Por último, puedes comenzar a escribir las clases de utilidad de ***Tailwind*** en tu aplicación.

`app/page.tsx`

```tsx
export default function Page() {
    return <h1 className="text-3xl font-bold underline">Hello, Next.js!</h1>
}
```

## Sass

***Next.js*** se integra con ***Sass*** utilizando las extensiones y sintaxis `.scss` y `.sass`.

También puedes utilizar ***Sass*** a nivel de ***componente*** a través de módulos ***CSS*** y la extensión `.module.scss` o `.module.sass`.

### Instalación de Sass

Para comenzar a utilizar ***Sass***, instale el paquete `sass`:

```bash
npm install --save-dev sass
```

### Personalización de las opciones de ***Sass***

Si desea configurar sus opciones de ***Sass***, utilice la opción `sassOptions` en `next.config.js`.

`next.config.js`

```tsx
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
    sassOptions: {
        additionalData: `$var: red;`,
    },
}

export default nextConfig
```

## CSS en JS

⚠️

Advertencia:

Las bibliotecas ***CSS-in-JS*** que requieren ***JavaScript*** en tiempo de ejecución actualmente no son compatibles con los componentes de servidor de ***React***. El uso de ***CSS-in-JS*** con funciones de ***React*** más nuevas, como los componentes de servidor y la transmisión, requiere que los autores de bibliotecas admitan la última versión de ***React***.

</aside>

Las siguientes bibliotecas son compatibles con los componentes del cliente en el directorio de la aplicación (en orden alfabético):

- `ant-design`
- `chakra-ui`
- `@fluentui/react-components`
- `kuma-ui`
- `@mui/material`
- `@mui/joy`
- `pandacss`
- `styled-jsx`
- `styled-components`
- `stylex`
- `tamagui`
- `tss-react`
- `vanilla-extract`

Actualmente se encuentran trabajando en brindar soporte los siguientes:

`emotion`

Si desea diseñar componentes de servidor, le recomendamos utilizar módulos ***CSS*** u otras soluciones que generen archivos ***CSS***, como ***Tailwind CSS***.

### Configuración de CSS en JS

Para configurar ***CSS-in-JS***, necesitas:

1. Crea un registro de estilos para recopilar todas las reglas ***CSS*** en una representación.
2. Utilice el ***hook*** `useServerInsertedHTML` para inyectar reglas antes de cualquier contenido que pueda usarlas.
3. Cree un componente de cliente que envuelva su aplicación con el registro de estilos durante la representación inicial del lado del servidor.

#### styled-jsx

Para configurar `styled-jsx` para su aplicación, cree un nuevo registro:

`app/registry.tsx`

```tsx
'use client'

import React, { useState } from 'react'
import { useServerInsertedHTML } from 'next/navigation'
import { StyleRegistry, createStyleRegistry } from 'styled-jsx'

export default function StyledJsxRegistry({
    children,
}: {
    children: React.ReactNode
}) {
  // Only create stylesheet once with lazy initial state
  // x-ref: https://reactjs.org/docs/hooks-reference.html#lazy-initial-state
    const [jsxStyleRegistry] = useState(() => createStyleRegistry())
    
    useServerInsertedHTML(() => {
        const styles = jsxStyleRegistry.styles()
        jsxStyleRegistry.flush()
        return <>{styles}</>
    })
    
    return <StyleRegistry registry={jsxStyleRegistry}>{children}</StyleRegistry>
}
```

Luego, envuelva el ***root layout*** con el registro:

`app/layout.tsx`

```tsx
import StyledJsxRegistry from './registry'

export default function RootLayout({
    children,
}: {
    children: React.ReactNode
}) {
    return (
        <html>
            <body>
                <StyledJsxRegistry>{children}</StyledJsxRegistry>
            </body>
        </html>
    )
}
```

#### styled-components

Para usar componentes con estilo, habilítelos en `next.config.js`:

next.config.ts

```tsx
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
    compiler: {
        styledComponents: true,
    },
}

export default nextConfig
```

Luego, use la ***API styled-components*** para crear un componente de registro global para recopilar todas las reglas de estilo ***CSS*** generadas durante una representación y una función para devolver esas reglas. Luego, use el ***hook*** `useServerInsertedHTML` para inyectar los estilos recopilados en el registro en la etiqueta ***HTML*** `<head>` en el ***root layout***.

`lib/registry.tsx`

```tsx
'use client'

import React, { useState } from 'react'
import { useServerInsertedHTML } from 'next/navigation'
import { ServerStyleSheet, StyleSheetManager } from 'styled-components'

export default function StyledComponentsRegistry({
    children,
}: {
    children: React.ReactNode
}) {
  // Only create stylesheet once with lazy initial state
  // x-ref: https://reactjs.org/docs/hooks-reference.html#lazy-initial-state
    const [styledComponentsStyleSheet] = useState(() => new ServerStyleSheet())
    
    useServerInsertedHTML(() => {
        const styles = styledComponentsStyleSheet.getStyleElement()
        styledComponentsStyleSheet.instance.clearTag()
        return <>{styles}</>
    })

    if (typeof window !== 'undefined') return <>{children}</>

    return (
        <StyleSheetManager sheet={styledComponentsStyleSheet.instance}>
            {children}
        </StyleSheetManager>
    )
}
```

Envuelva a los hijos del ***root layout*** con el componente de registro de estilo:

`app/layout.tsx`

```tsx
import StyledComponentsRegistry from './lib/registry'

export default function RootLayout({
    children,
}: {
    children: React.ReactNode
}) {
    return (
        <html>
            <body>
                <StyledComponentsRegistry>{children}</StyledComponentsRegistry>
            </body>
        </html>
    )
}
```

## Hojas de estilo externas

Las hojas de estilo publicadas por paquetes externos se pueden importar en cualquier lugar del directorio de la aplicación, incluidos los componentes ubicados en el mismo lugar:

`app/layout.tsx`

```tsx
import 'bootstrap/dist/css/bootstrap.css'

export default function RootLayout({
    children,
}: {
    children: React.ReactNode
}) {
    return (
        <html lang="en">
            <body className="container">{children}</body>
        </html>
    )
}
```

Las hojas de estilo externas se deben importar directamente desde un paquete `npm` o descargar y colocar junto con el código base. No se puede usar `<link rel="stylesheet" />`.
