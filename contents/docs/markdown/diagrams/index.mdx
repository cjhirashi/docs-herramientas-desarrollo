Los diagramas son herramientas poderosas para visualizar procesos, relaciones y decisiones. Esta sección muestra diferentes tipos de diagramas creados con **Mermaid**, junto con ejemplos y fragmentos de código reutilizables para integrar en sus proyectos.

## Diagrama de flujo

Un diagrama de flujo representa una secuencia de pasos o procesos en formato visual. Utilice este diagrama para representar gráficamente flujos de trabajo, procesos de toma de decisiones o flujos operativos.

<Mermaid
  chart={`graph TD;
    Start --> Task1;
    Task1 --> Task2;
    Task2 --> End;`}
/>

```jsx
<Mermaid
  chart={\`
    graph TD;
    Start --> Task1;
    Task1 --> Task2;
    Task2 --> End;
  \`}
/>
```

## Árbol de decisión

Los árboles de decisión ilustran opciones y posibles resultados, lo que los hace ideales para flujos de trabajo de toma de decisiones o procesos que involucran múltiples caminos.

<Mermaid
  chart={`graph TD;
    A[Start] --> B{Is it raining?};
    B -->|Yes| C[Take an umbrella];
    B -->|No| D[Enjoy the weather];
    C --> E[Go outside];
    D --> E;`}
/>

```jsx
<Mermaid
  chart={\`
    graph TD;
    A[Start] --> B{Is it raining?};
    B -->|Yes| C[Take an umbrella];
    B -->|No| D[Enjoy the weather];
    C --> E[Go outside];
    D --> E;
  \`}
/>
```

## Diagrama de entidad-relación

Los diagramas de relación entre entidades (ERD) se utilizan para modelar las relaciones entre entidades de un sistema. Se utilizan ampliamente en el diseño de bases de datos y en la planificación de la arquitectura de sistemas.

<Mermaid
  chart={`erDiagram
    CUSTOMER ||--o{ ORDER : places
    ORDER ||--|{ LINE-ITEM : contains
    PRODUCT ||--o{ LINE-ITEM : "included in"
    CUSTOMER {
        string name
        string email
    }
    ORDER {
        int orderNumber
        date orderDate
    }
    LINE-ITEM {
        int quantity
        float price
    }
    PRODUCT {
        int productId
        string name
        float price
    }`}
/>

```jsx
<Mermaid
  chart={\`
    erDiagram
    CUSTOMER ||--o{ ORDER : places
    ORDER ||--|{ LINE-ITEM : contains
    PRODUCT ||--o{ LINE-ITEM : "included in"
    CUSTOMER {
        string name
        string email
    }
    ORDER {
        int orderNumber
        date orderDate
    }
    LINE-ITEM {
        int quantity
        float price
    }
    PRODUCT {
        int productId
        string name
        float price
    }
  \`}
/>
```

Cada uno de estos diagramas cumple una función específica y Mermaid facilita su generación dinámica. Experimente con los fragmentos de código proporcionados y adáptelos a sus necesidades.
