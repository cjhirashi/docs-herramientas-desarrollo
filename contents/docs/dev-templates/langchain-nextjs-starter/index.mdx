## DESCRIPCION GENERAL 

Esta plantilla es el andamiaje de una aplicación de inicio LangChain.js + Next.js. Muestra cómo usar y combinar módulos LangChain para varios casos de uso. Específicamente:

* Chat simple
* Cómo devolver un resultado estructurado de una llamada LLM
* Cómo responder preguntas complejas de varios pasos con agentes
* Recuperación de generación aumentada (RAG) con una cadena y un almacén de vectores
* Recuperación de generación aumentada (RAG) con un agente y un almacén de vectores

### STACK TECNOLOGICO

* **Framework**: Next.js
* **Caso de uso**: AI
* **CSS**: Tailwind
* **Database**: Supabase

### CARACTERISTICAS

La mayoría de ellos utilizan el Vercel AI SDK para transmitir tokens al cliente y mostrar los mensajes entrantes.

Los agentes utilizan ***LangGraph.js***, el marco de LangChain para crear flujos de trabajo de agentes. Utilizan funciones auxiliares preconfiguradas para minimizar el código repetitivo, pero puedes reemplazarlas con gráficos personalizados según lo desees.

## CLONAR REPOSITORIO 

<Step>
  <StepItem title="Clonar repositorio">
   
    Clona el repositorio de GitHub y accede a la nueva carpeta del proyecto

    ```bash
    git clone https://github.com/langchain-ai/langchain-nextjs-template.git
    cd langchain-nextjs-template
    ```

  </StepItem>

  <StepItem title="Conectar con nuevo repositorio de GitHub">
    
    1. Crea un nuevo repositorio en [GitHub](https://github.com/)
    2. Verificar qué origen tiene asignado el repositorio del proyecto clonado
        ```bash
        git remote -v
        ```
    2. Ejectura el comando de reasignación de Origen del proyecto con el siguiente comando:
        ```bash
        git remote set-url origin [URL de nuevo repositorio]
        ```
  </StepItem>

  <StepItem title="Configuración">
    
    A continuación, deberá configurar las variables de entorno en el archivo `.env.local` de su repositorio. Copie el archivo `.env.example` en `.env.local`. Para comenzar con los ejemplos básicos, solo deberá agregar su clave API de OpenAI.

    Debido a que esta aplicación está diseñada para ejecutarse en funciones de Edge sin servidor, asegúrese de haber configurado la variable de entorno LANGCHAIN_CALLBACKS_BACKGROUND en falso para garantizar que finalice el seguimiento si está utilizando el seguimiento de LangSmith.

    A continuación, instale los paquetes necesarios utilizando su administrador de paquetes preferido (por ejemplo, yarn).

  </StepItem>
  
  <StepItem title="Instalar dependencias">
    
    Instala todas las dependencias del proyecto
    
    ```bash
    yarn install
    ```
  </StepItem>

  <StepItem title="Ejecutar servidor en desarrollo">
    
    Ejecutar el servidor en modo desarrollo
    
    ```bash
    yarn run dev
    ```
  </StepItem>
</Step>
